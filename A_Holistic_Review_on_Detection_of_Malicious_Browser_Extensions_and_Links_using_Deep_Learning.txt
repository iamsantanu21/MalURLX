Abstract—The growth of the Internet has aroused people’s
attention toward network security. A secure network environment
is fundamental for the expeditious and impeccable development
of the Internet. The majority of internet-based tasks can be
completed with the help of a web browser. Although many web
applications add browser extensions to improve their functional-
ity, some of these extensions are malicious and can access sensitive
data without the user’s knowledge. Browser extensions with mali-
cious intent present a growing security concern and have quickly
become one of the most prevalent methods used to compromise
Internet security. This is largely due to their widespread usage
and the extensive privileges they possess. After being installed,
these malicious extensions are executed and make an attempt to
compromise the victim’s browser. This makes them particularly
elusive and challenging to combat. It is crucial to promptly
develop an effective strategy to address the threats posed by these
extensions. A comprehensive review of the research on browser
extension vulnerabilities is presented in this paper. The role of
malicious links in web browser extensions are examined for sev-
eral attacks. Detection of malicious browser extension on various
aspects are represented namely Intrusion malicious web browser
extensions detection using Intrusion detection, Machine learning
based detection methods and Deep learning based techniques to
mitigate malicious web browser extensions are examined. This
study investigates the critical function of malicious detection in
protecting web browsers, looking at the changing threats and
risk-reduction tactics. A robust cybersecurity frameworks can be
created that not only respond to known threats but also anticipate
and thwart the strategies of future cyber adversaries by realizing
the significance of proactive detection. Thus this survey provides
a detailed comparison of various solutions for malicious browser
extension.
Index Terms —Cyber-attacks, Machine learning, Malicious
Browser extension, and Malicious Uniform Resource Locator
(URL).
I. I NTRODUCTION
Cybersecurity is the discipline of defending data, networks,
and computer systems against damage, attacks, and unau-
thorized access. It includes a variety of tools, procedures,
and controls intended to protect digital data and guarantee
the privacy, availability, and integrity of vital resources [1].
The need for strong cybersecurity strategies to combat these
ever-growing challenges is highlighted by the fact that cyber
threats, which range from ransomware and phishing attacks
to advanced persistent threats, are becoming more and more
sophisticated due to the increased sophistication of malicious
actors.
The different kinds of cyber incidents and their effects on
different industries [2] are depicted in the Figure 1. People
can engage with all the information on the Internet via web
browsers, such as Internet Explorer, Opera, Firefox, Safari,
and Chrome. To make them more functional, the extensions
are permitted to leverage browser features, such as browser
history, bookmarks, webpage resources, and favorite social
features, to enhance and personalize the user’s surfing experi-
ence. Users can download a variety of extensions from third-
party websites or from the official storefronts of the browser
providers, such as Opera Add-Ons, Safari Extensions Gallery,
and Chrome Web Store [3].
Threats posed by web browser extensions to privacy are
a serious cybersecurity issue because they increase attack
surfaces and make different types of exploits possible [4].
These threats—which include malware distribution, phishing,
resource exploitation, and unauthorized access—highlight the
necessity of implementing thorough security measures. In
order to mitigate these risks and protect against ever-evolving
cyber threats, careful extension vetting, user education, and
proactive security strategies are necessary.

IV. DETECTION OF MALICIOUS BROWSER
EXTENSION BASED ON MACHINE LEARNING
TECHNIQUES
In [19], the authors propose a novel method for classifying
webpages based on keyword density and context using three
supervised machine learning techniques: maximum entropy,
extreme learning machine, and support vector machine. In [20]
employed C4.5 as the classification algorithm after analyzing
visual brand entities such as the favicon, logo, and copyright
notice in phishing websites, extracting brand features and
brand authorization features. In order to distinguish the site
with branding rights from phishing sites, incoming links,
redirection, and DNS information-based brand authorization
features are further extracted to address this issue. This [21]
created a feature for target identification that can recognize the
target website that a phishing webpage is trying to imitate. This
phishing detection system’s target detection component can
reduce false positives and operates faster than previously doc-
umented systems. The feature set incorporates inherent phisher
limitations, making the system resistant to adaptive attackers
who modify phishes to evade detection. The fundamental
phishing detector component is quick and does not require
internet access to centralized data. As such, it is ideal for
a client-side implementation that respects privacy. The target
brand identification component of this model can extract a set
of keyterms that describe a webpage using a straightforward
technique. If the webpage is a phish, the model use the
keyterms set to identify its target. This work [22] incorporated
optical character recognition for information extraction from
the images and remaining other potential helpful information.
This work [23] aims to develop a Chrome extension that will
operate as a go-between for users and malicious websites,
reducing the likelihood that users will fall victim to them.
Furthermore, since harmful content is constantly evolving, it
is impossible to compile all of it. In order to combat this,
the authors employed machine learning to train the tool and
classify newly discovered content into specific categories each
time, allowing appropriate action to be performed. SVM was
utilized by in [24] to find malicious URLs on Facebook. In
[25], the authors used Random Forest as the classifier after
extracting features from user-provided URLs and source code.

C. Deep Learning Process
Deep Learning (DL) represents the following steps namely
data collection, data preprocessing, model training and model
evaluation [7].
1) Data Preprocessing: The initial step in machine learning
and DL task is data preprocessing. It gets ready-made text,
image, and other raw data ready for model classification.
The data preprocessing used for URL-based and hybrid-based
phishing detection is examined in this section. Three steps are
involved in working with raw data: data cleaning, tokenization,
and representation. Data cleaning first eliminates extraneous
information from the input, such as special characters, stop-
words, and data noise. On the other hand, a URL has special
characters that stand for important information. For instance,
a URL fragment is represented by the hash symbol ”#.” As
a result, it’s crucial to remove special characters carefully to
avoid losing any crucial information from the URL. Therefore,
the majority of URL-based models, including [37], [38], [39],
and [40], do not require data cleaning.
Tokenization is an essential step in the data preprocessing
process of deep learning. Tokenization is the process of
dividing a text into smaller units called tokens, which are
usually words or subwords. Through this procedure, raw text
data can be transformed into a format that deep learning
models can comprehend and handle efficiently. The input
data for the URL-based can be divided into character-level
segments, like [37], [39], and [41]. It can also be divided
into words, like in [42]. Additionally, some papers—like
[38], [40] and [43]—combine word-level and character-level
analysis. Using URL symbols like ”.,” ”:,” ”//,” and so forth,
one can divide URLs into word-level segments. For model
training in deep learning data preprocessing, effective data
representation is crucial. Wordembedding for natural language
processing, one-hot encoding for categorical variables, and
normalization or standardization for numerical features are
examples of common techniques. Sequence padding guaran-
tees that the input shapes are consistent, and preprocessing for
image data includes augmentation, normalization, and resizing.
An increasingly stable dataset is a result of feature scaling
and missing data management. The type of data and the
particular requirements of the deep learning task determine
which representation is best.
2) Deep Learning Modeling: Deng and Yu [44] di-
vide deep learning techniques into two categories: gener-
ative/unsupervised models and deep discriminative models.
Three methods are used in the deep discriminative mod-
els: convolutional neural networks, deep neural networks,
and recurrent neural networks. Deep autoencoders, restricted
Boltzmann machines, deep Boltzmann machines, and deep
belief networks are the four generative/unsupervised model
approaches. The broadly used deep learning algorithms for
detecting malicious web browser extensions are as follows:
a) CNN:: Convolutional neural networks (CNNs) are a
popular feedforward deep learning algorithm for classifying

VI. OPPORTUNITIES & CHALLENGES
The research on the mitigation of web browser attacks is
constantly evolving. However, there are still challenges or
drawbacks in web browser extension solutions.
A. Optimal Quality datasets
The size and validity of the training dataset have a signif-
icant impact on the performance of machine learning-based
solutions. The power law states that as the size of the training
data increases, deep learning performance increases as well.
This is contingent upon the stability of the websites or services
provided by third parties [48].
B. Strategic feature extraction & selection techniques
Feature extraction and selection faces two difficulties
namely: High-dimensional input spaces and Data variabil-
ity. The former still have challenges associated with high-
dimension data and the curse of dimensionality and there
is a need for dimensionality reduction strategies. In later, it
examines the difficulties caused by the distribution of data’s
variability and it represents the impact of noisy and irrelevant
features on model performance [49].
C. Tiny URL detection
Due to the widespread usage of URL shortening services
for link sharing, TinyURLs present a serious cybersecurity
risk since they give attackers the chance to conceal the
destination of malicious links. To effectively address the risks
associated with malicious TinyURLs, a multifaceted approach
combining collaborative threat intelligence, machine learning,
and behavioral analysis is essential [50].
D. Real time systems and its effective responses
Rule-based models rely on third-party services from a URL
string and rule parsing. As such, in a real-time prediction
system that takes a single URL string as an input in every
client request, they require a comparatively long response
time [50]. Moreover, rule based model highly depend on
heuristics and patterns which makes them least adaptable to
cyber threats.

addressing cybersecurity challenges, specifically in the context
of web browser extension attacks. It thoroughly examines
how browser extensions can be manipulated to spread ma-
licious content, underscoring the security risks they pose.
The investigation reveals the sophisticated methods by which
extensions can serve as vectors for cyber threats, including the
dissemination of harmful links. The survey delves into various
cybersecurity risks and explores detection mechanisms, with
a focus on the novel integration of malicious link analysis
into the exploitation of web browser extensions. This demon-
strates how attacks can escalate through the combined use of
malicious links and browser extensions. The survey analyzes
different strategies to mitigate these risks, spotlighting the role
of intrusion detection, machine learning, and deep learning
techniques. It reviews extensive literature on the stages of DL
processes and modeling, presenting efficient methodologies
to combat cyber threats. The challenges in detecting and
mitigating these risks are discussed, along with the necessity of
advanced DL techniques to enhance online security. Finally,
the open challenges and future directions to mitigate mali-
cious web browser extensions are investigated. This survey is
capable to elicit further discussion and research on dynamic
defense to deter malicious web browser extensions.
